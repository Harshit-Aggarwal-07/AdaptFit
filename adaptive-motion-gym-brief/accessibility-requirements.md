# Accessibility Requirements

The Adaptive Motion Gym module should be accessibility-first.

## Required

- Keyboard navigation
- Semantic HTML where applicable
- Clear focus states
- Screen-reader-friendly labels
- High contrast mode
- Large text mode
- Reduced motion mode
- Text alternatives for visual guides
- No color-only information
- Accessible form labels
- Clear error messages
- Responsive mobile-first design
- Voice-ready workout instructions
- Captions/text instructions for movement
- Buttons with descriptive names

## Accessibility Panel

Include controls for:

- High contrast
- Large text
- Reduced motion
- Dyslexia-friendly font if possible
- Simple language if possible
- Voice guidance
- Captions/text instructions

## Acceptance Checks

- Can the user complete onboarding with keyboard only?
- Can the user generate a workout without a mouse?
- Are all form fields labeled?
- Are all buttons understandable out of context?
- Are visual exercise steps described in text?
- Does reduced motion reduce animations?
- Does high contrast improve readability?
- Does large text work across major screens?
